5. DETECTION ENGINEERING (VERY DEEP)
------------------------------------------------------------------------------------------------------

DETECTION TYPES:

    1. Signature-Based:
        -> Known attack patterns
        -> Example: Malware hash

    2. Behavior-Based:
        -> Abnormal activity
        -> Example: Unusual login time

    3. Anomaly-Based:
        -> ML deviations from baseline

RULE CREATION PIPELINE:

    Threat Intel --> MITRE Mapping --> Rule Logic --> Testing --> Deployment

EXAMPLE RULE (ASCII LOGIC):

    IF:
        Failed_Logins > 10
        AND Source_IP != Known
    THEN:
        Trigger Alert: "Brute Force Attack"

MITRE ATT&CK MAPPING:

    +------------+----------------------------+
    | Tactic     | Technique                  |
    +------------+----------------------------+
    | Initial    | Phishing                   |
    | Execution  | PowerShell                 |
    | Persistence| Registry Run Keys          |
    | Lateral    | Pass-the-Hash              |
    +------------+----------------------------+
